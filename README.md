## Architecture

![Architecture Diagram](docs/architecture.png)

```
Browser
   │
   ▼
Streamlit Frontend  (port 8501)
   │  HTTP POST /chat
   ▼
FastAPI Backend     (port 8000)
   │  boto3
   ▼
Amazon Bedrock Agent Runtime
Retrieve API
   │
   ├── Semantic Search ──► Knowledge Base
   └── Answer Generation ► Anthropic Claude 3 Sonnet
```

### Request flow

1. User types a question in the Streamlit UI
2. Streamlit POSTs `{ "question": "..." }` to FastAPI `/chat`
3. FastAPI validates the request via Pydantic v2
4. `BedrockService` calls the Amazon Bedrock Retrieve API to fetch the most relevant document chunks
5. Bedrock performs semantic search over the Knowledge Base
6. Claude 3 Sonnet generates a grounded answer from retrieved context
7. Answer is returned through FastAPI → Streamlit → User

## Local Development Setup

### 1. Clone the repository

```bash
git clone https://github.com/gani1123/aws-devops-knowledge-assistant.git
cd aws-devops-knowledge-assistant
```

### 2. Backend setup

```bash
cd backend
python3.12 -m venv .venv
source .venv/bin/activate        # macOS / Linux
.venv\Scripts\activate           # Windows

pip install --upgrade pip
pip install -r requirements.txt

cp .env.example .env             # then edit .env with your values
```

### 3. Frontend setup

```bash
cd frontend
python3.12 -m venv .venv
source .venv/bin/activate

pip install --upgrade pip
pip install -r requirements.txt
```

### 4. Configure AWS credentials

```bash
aws configure
# --- OR ---
# Attach an IAM Role to your EC2 / ECS instance (recommended for production)
```

### 5. Run the backend

```bash
cd backend
uvicorn app:app --reload --host 0.0.0.0 --port 8000
```

### 6. Run the frontend

```bash
cd frontend
streamlit run streamlit_app.py
```

| Service | URL |
|---|---|
| FastAPI Backend | http://localhost:8000 |
| Swagger UI | http://localhost:8000/docs |
| Streamlit Frontend | http://localhost:8501 |

## Running with Docker

### Build and start the full stack

```bash
docker compose -f docker/docker-compose.yml up --build
```

### Run in detached mode

```bash
docker compose -f docker/docker-compose.yml up --build -d
```

### View logs

```bash
# All services
docker compose -f docker/docker-compose.yml logs -f

# Backend only
docker compose -f docker/docker-compose.yml logs -f backend

# Frontend only
docker compose -f docker/docker-compose.yml logs -f frontend
```

### Stop the stack

```bash
docker compose -f docker/docker-compose.yml down
```

> **Production note:** In production (ECS, EC2 with IAM Role), remove the `~/.aws` volume mount from `docker-compose.yml`. The boto3 default credential chain will automatically pick up the instance metadata credentials.

## API Reference

### `GET /`

API identification endpoint.

**Response 200**
```json
{ "message": "AWS DevOps Knowledge Assistant API" }
```

### `GET /health`

Liveness probe for load balancers and container orchestrators.

**Response 200**
```json
{ "status": "healthy" }
```

### `POST /chat`

Submit a question to the Bedrock Knowledge Base.

**Request**
```json
{ "question": "What is IAM?" }
```

| Field | Type | Required | Constraints |
|---|---|---|---|
| `question` | string | ✅ | 3 – 1000 characters |

**Response 200**
```json
{
  "answer": "IAM stands for Identity and Access Management..."
}
```

**Response 503** — Bedrock unavailable
```json
{ "detail": "Knowledge Base service error: ..." }
```

**Response 422** — Validation error
```json
{ "detail": [{ "loc": ["body", "question"], "msg": "...", "type": "..." }] }
```

## AWS Configuration

| Parameter | Value |
|---|---|
| Region | `us-east-1` |
| Bedrock Runtime API | `bedrock-agent-runtime` |
| Foundation Model | `anthropic.claude-3-sonnet-20240229-v1:0` |

> **Note:** The Knowledge Base ID is configured via the `KNOWLEDGE_BASE_ID` environment variable. Do not hardcode it — set it in your `.env` file or pass it as an environment variable in your deployment.

### Required IAM permissions

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "bedrock:RetrieveAndGenerate",
        "bedrock:Retrieve",
        "bedrock:InvokeModel"
      ],
      "Resource": "*"
    }
  ]
}
```

## Environment Variables

Copy `backend/.env.example` to `backend/.env` and fill in your values.

| Variable | Default | Description |
|---|---|---|
| `APP_NAME` | `AWS DevOps Knowledge Assistant` | Application display name |
| `APP_VERSION` | `1.0.0` | Semantic version string |
| `DEBUG` | `false` | Enables debug logging and reload |
| `HOST` | `0.0.0.0` | Uvicorn bind address |
| `PORT` | `8000` | Uvicorn listen port |
| `AWS_REGION` | `us-east-1` | AWS region for Bedrock |
| `KNOWLEDGE_BASE_ID` | *(required)* | Your Bedrock Knowledge Base ID |
| `BEDROCK_MODEL_ARN` | *(required)* | Full ARN of the foundation model |
| `LOG_LEVEL` | `INFO` | Python logging level |

## Logging

All logs are written to stdout in a structured format compatible with AWS CloudWatch, ECS, and any log aggregation platform:

```
2024-01-15T10:23:45 | INFO     | api.routes | POST /chat | question_length=18
2024-01-15T10:23:46 | INFO     | services.bedrock_service | Received answer from Bedrock KB | answer_length=312
```

Set `LOG_LEVEL=DEBUG` for verbose request-level tracing during development.
